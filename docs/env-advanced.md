# Advanced configuration reference

The slim `env.example` covers the variables 95% of installs need. This page lists every environment variable the backend reads, grouped by subsystem. Add the ones you want to `~/.synsci/delphi/source/.env` and run `delphi reload` (or restart api + worker manually).

For interactive editing of the install-time choices (provider, model, keys, dashboard password) use `delphi config` instead of editing `.env` by hand.

> **Heads-up:** this page is hand-curated and updated on backend releases. The source of truth is `backend/synsc/config.py` and `backend/synsc/api/http_server.py`.

---

## Dashboard auth

| Var | Default | Description |
|---|---|---|
| `SYSTEM_PASSWORD` | — | Dashboard login + bootstrap secret for minting API keys. Required. |
| `SERVER_SECRET` | — | JWT signing key. Auto-generated by installer; rotating invalidates every existing session token. |
| `TOKEN_ENCRYPTION_KEY` | — | Optional Fernet key encrypting stored OAuth/HF tokens at rest. Without it, tokens are stored plaintext (in DB, not in env). |
| `GITHUB_CLIENT_ID` | — | Enable "Sign in with GitHub" on the dashboard login page. Pair with `GITHUB_CLIENT_SECRET`. |
| `GITHUB_CLIENT_SECRET` | — | GitHub OAuth secret. Don't ship this in any redistributed package — bake nothing, ask users to register their own OAuth app. |
| `SYNSC_REQUIRE_AUTH` | `true` | When `false`, the api accepts unauthenticated calls. Only useful in trusted single-user dev environments. |
| `SYNSC_COOKIE_SECURE` | auto | Force `Secure` flag on the session cookie. Default auto-detects HTTPS. |
| `SYNSC_ENABLE_HSTS` | `false` | Send Strict-Transport-Security headers. Enable when running behind HTTPS. |

## Embeddings

### Common

| Var | Default | Description |
|---|---|---|
| `EMBEDDING_PROVIDER` | `local` | One of `local` / `gemini` / `openai` / `huggingface`. The CLI's `delphi config` covers the first three; `huggingface` (HF Inference API) is set manually. |
| `EMBEDDING_MODEL` | `BAAI/bge-base-en-v1.5` | Model id. For `local` and `huggingface` this is any HF repo id. For `gemini` and `openai` it's overridden to a fixed model. |
| `EMBEDDING_DIMENSION` | `768` | Vector size. **Do not change without migrating the pgvector schema** — the column is `vector(768)`. |
| `EMBEDDING_DEVICE` | `auto` | Local provider only. `cpu`, `cuda`, `mps`, or `auto`. |
| `EMBEDDING_BATCH_SIZE` | `64` | Local + remote batch size. Bigger batches → faster indexing on GPU, more memory pressure. |

### HuggingFace

| Var | Default | Description |
|---|---|---|
| `HF_TOKEN` | — | HuggingFace API token. Required for gated/private models on the Local provider; required for the HuggingFace Inference API provider; optional otherwise (raises Hub download rate limits). |
| `HF_INFERENCE_ENDPOINT` | — | Override the HF Inference URL. Set when you've deployed a model to a paid HF Inference Endpoint. |

### OpenAI

| Var | Default | Description |
|---|---|---|
| `OPENAI_API_KEY` | — | Required when `EMBEDDING_PROVIDER=openai`. |
| `OPENAI_EMBEDDING_MODEL` | `text-embedding-3-small` | Override the OpenAI embedding model. `text-embedding-3-large` is ~6× more expensive but a few percentage points better on retrieval benchmarks. |

### Google Gemini

| Var | Default | Description |
|---|---|---|
| `GEMINI_API_KEY` | — | Required when `EMBEDDING_PROVIDER=gemini`. Also doubles as the research-provider key (see Research below). |

## Reranker (post-retrieval re-ranking)

| Var | Default | Description |
|---|---|---|
| `SYNSC_ENABLE_RERANKER` | `false` | Enable cross-encoder re-rank on the top-K candidates. Improves precision; costs a forward pass per query. |
| `RERANKER_MODEL` | `cross-encoder/ms-marco-MiniLM-L-6-v2` | sentence-transformers cross-encoder model id. |
| `RERANKER_BLEND_ALPHA` | `0.5` | Mix between vector similarity and reranker score. `0` = pure reranker, `1` = pure vector. |

## Research (RAG synthesis via Gemini)

| Var | Default | Description |
|---|---|---|
| `SYNSC_RESEARCH_PROVIDER` | `gemini` | Backend for the `research` MCP tool. Currently only `gemini` is implemented. |
| `SYNSC_RESEARCH_API_KEY` | falls back to `GEMINI_API_KEY` | Provider key. Set to use a different key from the embeddings one. |
| `SYNSC_RESEARCH_MODEL_QUICK` | `gemini-2.5-flash-lite` | Model for `mode=quick` calls. |
| `SYNSC_RESEARCH_MODEL_DEEP` | `gemini-2.5-pro` | Model for `mode=deep` and `mode=oracle`. |

## Rate limits

Buckets are per-(api-key, mode) sliding windows (in-process; not durable across restarts). Disable globally with `SYNSC_RATE_LIMIT_ENABLED=false` — only safe in trusted single-user environments.

| Var | Default | Description |
|---|---|---|
| `SYNSC_RATE_LIMIT_ENABLED` | `true` | Master switch. |
| `SYNSC_RATE_LIMIT_DEFAULT` | `60/minute` | Default quota for endpoints without a specific tag. |
| `SYNSC_RATE_LIMIT_AUTH` | `10/minute` | `/auth/*` endpoints. |
| `SYNSC_RATE_LIMIT_INDEX` | `5/minute` | Indexing endpoints (repos, papers, datasets). |
| `SYNSC_RATE_LIMIT_SEARCH` | `30/minute` | Search endpoints. |

## Picking values

Quick recipes for common changes:

**Speed up indexing (big-RAM machine):**
```ini
EMBEDDING_BATCH_SIZE=256
SYNSC_WORKER_THREADS=8
```

**Better search quality (latency budget allows):**
```ini
SYNSC_ENABLE_RERANKER=true
RERANKER_BLEND_ALPHA=0.4
```

**Run on GPU:**
```ini
EMBEDDING_DEVICE=cuda
EMBEDDING_BATCH_SIZE=128
```

**Run dashboard on a custom hostname:**
```ini
FRONTEND_URL=https://delphi.mycompany.internal
SYNSC_CORS_ORIGINS=https://delphi.mycompany.internal
SYNSC_COOKIE_SECURE=true
SYNSC_ENABLE_HSTS=true
```

After any change, `delphi reload` to restart api + worker.
